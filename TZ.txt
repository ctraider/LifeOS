# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: РАЗРАБОТКА ИНТЕРФЕЙСА В СТИЛЕ RIMWORLD

## 1. ТЕКУЩЕЕ СОСТОЯНИЕ И ОБЩАЯ АРХИТЕКТУРА

### 1.1 Существующие элементы
- Главный экран с вкладками: Журнал, Вещи, Раб, Социум, Персона, Нужды, Здоровье
- Базовые индикаторы потребностей: Сытость, Настроение, Сон, Красота окружения, Комфорт, Свежий воздух
- Список мотиваторов настроения (положительные и отрицательные)
- Основная информация персонажа: имя, статус, фракция, возможность вербовки

### 1.2 Целевая архитектура
- Единая панель с обновленными вкладками:
  * Обзор (Summary)
  * Процедуры / История (Procedures / History)
  * Потребности (Needs) [переименование "Нужды"]
  * Социум (Social) [существует]
  * Персона (Persona) [существует]
  * Работа (Work) [переименование "Раб"]
  * Журнал (Journal) [существует]
  * Вещи [оценить необходимость - в ТЗ отсутствует]
- Контекстная панель справа для быстрого доступа к режимам (красная/зеленая иконки)

## 2. ПЛАН ПОЭТАПНОЙ РЕАЛИЗАЦИИ

### Этап 1: Реорганизация основной структуры
1. Переименовать вкладки согласно ТЗ:
   - "Нужды" → "Потребности"
   - "Раб" → "Работа"
2. Добавить отсутствующие вкладки:
   - "Обзор" (Summary)
   - "Процедуры/История" (Procedures/History)
3. Создать контекстную панель справа для переключения режимов

### Этап 2: Модуль "Здоровье" (Health)
1. Разработать головную панель управления:
   - Переключатель режима кормления (Allow food: Простая/Строгая)
   - Индикатор доступа к медикаментам (Allow medicine)
   - Опция самолечения (Allow self-tend) с индикацией разрешения
2. Создать список жизненных показателей:
   - Реализовать индикаторы: Боль, Сознание, Движение, Работа, Речь, Питание, Зрение, Слух, Дыхание, Фильтрация крови, Кровообращение, Пищеварение
   - Добавить процентное отображение и цветовую кодировку (зеленый/желтый/красный)
3. Внедрить систему травм и болезней:
   - Интерактивный список с информацией о локализации и статусе
   - Всплывающие окна с рекомендациями при клике на травму

### Этап 3: Доработка модуля "Потребности" (Needs)
1. Улучшить полосы состояния:
   - Добавить пороговые отметки на существующие индикаторы
   - Улучшить сегментацию полос
2. Расширить систему мотиваторов:
   - Доработать отображение стрелок и иконок изменения показателей
   - Реализовать всплывающие подсказки с детальной информацией

### Этап 4: Модуль "Социум" (Social)
1. Разработать таблицу отношений:
   - Список персонажей с индикаторами отношений
   - Показатели изменений за период (стрелки вверх/вниз)
2. Создать элементы управления социальными взаимодействиями:
   - Кнопка "Пригласить в группу"
   - Кнопка "Изгнать"
   - Фильтры отображения по типам отношений

### Этап 5: Расширение модуля "Персона" (Persona)
1. Дополнить информационный блок:
   - Улучшить отображение базовой информации (имя, пол, возраст, настроение, состояние)
   - Детализировать информацию о фракциях
2. Разработать систему черт характера:
   - Список trait-ов с описаниями
   - Визуализация модификаторов (положительных и отрицательных)
3. Внедрить систему XP для черт:
   - Отображение уровней (базовый, +10 XP, -5 XP)
   - Индикаторы прогресса

### Этап 6: Создание модуля "Работа" (Work)
1. Реализовать матрицу приоритетов:
   - Таблица заданий с возможностью установки уровней приоритетов (1-4)
   - Визуальное отображение текущих приоритетов
2. Добавить элементы управления:
   - Быстрые переключатели Вкл/Выкл для типов деятельности
   - Кнопка авто-распределения задач
3. Разработать алгоритмы:
   - Система авто-распределения на основе навыков
   - Учет текущих потребностей при распределении

### Этап 7: Доработка модуля "Журнал" (Journal)
1. Создать хронологию событий:
   - Список ключевых событий с временными метками
   - Категоризация событий (здоровье, соц.связи, работа)
2. Реализовать систему фильтрации:
   - Переключатели по типам событий
   - Временные фильтры (день, неделя, месяц)

### Этап 8: Новый модуль "Обзор" (Summary)
1. Разработать информационную панель:
   - Ключевые показатели персонажа
   - Графики состояния и прогресса
2. Создать систему оповещений:
   - Важные уведомления о критических состояниях
   - Рекомендации по улучшению показателей

### Этап 9: Новый модуль "Процедуры/История"
1. Реализовать интерфейс процедур:
   - Список доступных медицинских и социальных процедур
   - Кнопки быстрого назначения процедур
2. Создать исторический раздел:
   - Хронология значимых событий в жизни персонажа
   - Достижения и ключевые моменты

### Этап 10: Геймификация
1. Внедрить систему XP и уровней:
   - Визуализация прогресса выполнения задач
   - Индикаторы восстановления после травм
2. Реализовать систему достижений:
   - Ачивки за ключевые действия ("первое излечение", "100% сон ночью", "10 друзей")
   - Интерфейс отображения полученных достижений
3. Разработать механику виртуальной валюты:
   - Система накопления "кровяной" валюты
   - Интерфейс для отслеживания баланса и транзакций

## 3. ТЕХНИЧЕСКИЕ СПЕЦИФИКАЦИИ ДЛЯ РАЗРАБОТКИ

### 3.1 Общие требования к UI/UX
- Интерфейс должен соответствовать стилистике RimWorld
- Все элементы должны быть интуитивно понятны и легкодоступны
- Использовать единую цветовую кодировку: зеленый (хорошо), желтый (внимание), красный (проблема)

### 3.2 Требования к отзывчивости
- Все элементы управления должны мгновенно реагировать на действия пользователя
- Анимации переходов между вкладками должны быть плавными
- Информационные панели должны обновляться в реальном времени

### 3.3 Совместимость
- Интерфейс должен корректно масштабироваться под различные разрешения экрана
- Должна быть предусмотрена возможность настройки интерфейса под потребности пользователя
